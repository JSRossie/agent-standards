# References Section Format

Every document containing external citations must end with a **References** section structured as follows:

```markdown
## References

[1] J. Smith, "Article Title," *Publication*, Date. [Online]. Available: https://example.com. Accessed: Apr. 9, 2026.
[2] Organization Name, "Report Title," Report ID, Date. [Online]. Available: https://example.com. Accessed: Apr. 9, 2026.

---
*Citation format: IEEE ([style guide](https://ieee-dataport.org/sites/default/files/analysis/27/IEEE%20Citation%20Guidelines.pdf)) with project-specific business extensions*
```

## Rules

- References must be numbered sequentially in the order they first appear in the document. Do not alphabetize. Do not reuse numbers.
- If a source is removed during editing, renumber all subsequent references and update inline citations to match.
- Every References section must end with a horizontal rule (`---`) followed by the citation format footer line shown above. This footer is mandatory on every new document covered by this skill.
- One citation per line. Do not break a single citation across multiple lines.
- Leave one blank line between the last citation and the horizontal rule.
